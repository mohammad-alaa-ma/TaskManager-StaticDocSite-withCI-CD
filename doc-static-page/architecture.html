Application Architecture

Task Manager (3-Tier App)

  • Frontend (FE): Built with HTML, CSS, JavaScript. Dockerized and deployed as pods inside the cluster, exposed via a ClusterIP service, communicates with backend pods.
  • Backend (BE): Built with Node.js. Dockerized and deployed as pods inside the cluster, connects to external MySQL using its IP + port, exposed via a ClusterIP service.
  • MySQL Database: Runs outside Kubernetes on a dedicated machine, accessible at IP:3306.

Documentation Website

  • Runs as a static website deployment inside the cluster.
  • Serves this documentation page.
  • Exposed via a ClusterIP service and routed externally under /docs.

Deployment & Containerization

  • Each component (Frontend, Backend, Documentation) is dockerized and pushed to Docker Hub for easy deployment.
  • All dockerized services are orchestrated using Kubernetes for scalability and management.
  • ArgoCD is used for GitOps-based continuous deployment, syncing changes from Git & GitHub repositories.
  • The database remains on an external machine to ensure data persistence and separation of concerns.
